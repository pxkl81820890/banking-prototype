# Login Service

Multi-entity banking login service that handles user authentication with bank, branch, and currency context.

## Overview

The Login Service provides secure authentication for users in a multi-entity banking environment. It validates user credentials against specific bank and branch combinations, verifies currency context, and generates JWT tokens via the Authentication Service.

## Architecture

Built using Hexagonal Architecture (Ports & Adapters) with Domain-Driven Design principles:
- **Domain Layer**: Core business logic for login validation
- **Inbound Adapters**: REST API controllers
- **Outbound Adapters**: User repository, Authentication Service client

## Quick Start

### Prerequisites
- Java 17+
- Maven 3.8+
- Authentication Service running on port 8081

### Running the Service

```bash
mvn spring-boot:run
```

The service will start on `http://localhost:8080`

## API Endpoints

### POST /api/v1/auth/login

Authenticates a user with multi-entity context.

**Request:**
```bash
curl -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "bankCode": "101",
    "branchCode": "1119",
    "username": "john.doe",
    "password": "securePassword123",
    "currency": "SGD"
  }'
```

**Success Response (200 OK):**
```json
{
  "success": true,
  "userId": "user-uuid-123",
  "message": "Login successful",
  "token": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

**Error Responses:**

- **401 Unauthorized** - Invalid credentials
```json
{
  "timestamp": "2024-01-15T10:30:00Z",
  "status": 401,
  "error": "Unauthorized",
  "message": "Invalid credentials",
  "path": "/api/v1/auth/login"
}
```

- **404 Not Found** - Invalid entity (bank/branch/username combination)
```json
{
  "timestamp": "2024-01-15T10:30:00Z",
  "status": 404,
  "error": "Not Found",
  "message": "Invalid entity: Bank 101, Branch 1119 not found",
  "path": "/api/v1/auth/login"
}
```

- **400 Bad Request** - Currency mismatch
```json
{
  "timestamp": "2024-01-15T10:30:00Z",
  "status": 400,
  "error": "Bad Request",
  "message": "Currency mismatch: expected USD but got SGD",
  "path": "/api/v1/auth/login"
}
```

## Configuration

Key configuration properties in `application.yml`:

```yaml
server:
  port: 8080

app:
  authentication-service:
    url: http://localhost:8081
    timeout: 5s
```

## Security

- Passwords are hashed using BCrypt
- JWT tokens are generated by the Authentication Service
- Multi-entity validation ensures proper bank/branch context
- Currency validation prevents cross-currency authentication
